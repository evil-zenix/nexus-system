📍 БЫСТРАЯ СПРАВКА ПО КОМПОНЕНТАМ:

┌─────────────────────────────────────────────────────────────────────────────┐
│                        1. ROUTER (FastAPI :8000)                            │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Получает Webhook от Telegram                                              │
│ • Распарсивает JSON на основе bot_token из URL                              │
│ • Кидает сообщение в Redis очередь БЕЗ обработки                            │
│ • Отвечает Telegram мгновенно (200 OK)                                      │
│ • Масштабируется горизонтально (но на одном ПК достаточно 1 инстанса)       │
│                                                                              │
│ ⚡ Особенности:                                                             │
│   - Асинхронный (uvicorn + asyncio)                                         │
│   - Валидация JSON через Pydantic                                           │
│   - Health check endpoint для Docker                                        │
│   - Rate limiting (опционально)                                             │
│                                                                              │
│ 🔄 Поток данных: Telegram → Router → Redis → Workers                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                   2. REDIS (брокер сообщений + кэш)                         │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Очереди для каждого воркера: tasks:moderator, tasks:osint                │
│ • FIFO (First In First Out) логика                                          │
│ • Встроенная персистентность (RDB/AOF)                                      │
│ • LRU эвикция при заполнении (256MB лимит)                                  │
│ • Супер быстрые операции (< 1ms)                                            │
│                                                                              │
│ ⚡ Оптимизация:                                                             │
│   - Alpine образ (4MB вместо 80MB)                                          │
│   - Ограничение памяти (256MB)                                              │
│   - Отключены ненужные модули                                               │
│                                                                              │
│ 📊 Мониторинг:                                                              │
│   redis-cli> KEYS tasks:*        # список очередей                          │
│   redis-cli> LLEN tasks:osint    # количество задач                         │
│   redis-cli> INFO memory         # потребление памяти                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                    3. POSTGRESQL (основная БД)                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ Таблицы:                                                                    │
│   • users              (id, user_id, username, first_seen)                  │
│   • messages           (id, user_id, chat_id, text, timestamp)             │
│   • media_files        (id, file_id, media_type, size, telegram_url)       │
│   • moderation_logs    (id, message_id, is_spam, reason, timestamp)        │
│   • moderation_rules   (id, pattern, action, created_at)                   │
│                                                                              │
│ ⚡ Оптимизация:                                                             │
│   - Асинхронный драйвер (asyncpg)                                           │
│   - Connection pooling (минимум сокращен)                                   │
│   - max_connections=50                                                      │
│   - Индексы на часто используемых полях                                    │
│                                                                              │
│ 🔐 Безопасность:                                                            │
│   - Параметризованные запросы (защита от SQL инъекций)                     │
│   - Пароль в .env (не в коде)                                              │
│   - SQLAlchemy ORM (абстракция от БД)                                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                4. WORKER: MODERATOR (спам-проверка)                         │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Тянет задачи из Redis (tasks:moderator)                                   │
│ • Парсит сообщение (текст, URL, спам-паттерны)                              │
│ • Проверяет против правил модерации в БД                                    │
│ • Логирует результат в moderation_logs                                      │
│ • Опционально: вызывает Telegram API (delete message, ban user)             │
│                                                                              │
│ ⚡ Параметры:                                                               │
│   - WORKER_CONCURRENCY=5     # одновременно обрабатывает 5 задач           │
│   - QUEUE_NAME=tasks:moderator                                              │
│   - TASK_TIMEOUT=30          # сек на обработку одной задачи               │
│                                                                              │
│ 🔍 Примеры проверок:                                                        │
│   ✗ Много ссылок (spam links)                                               │
│   ✗ Запрещенные слова (blacklist)                                           │
│   ✗ Все капсом (flooding)                                                   │
│   ✗ Повторяющиеся символы (!!!, ???)                                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                5. WORKER: OSINT/SCRAPER (логирование)                       │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Тянет задачи из Redis (tasks:osint)                                       │
│ • Сохраняет текстовые сообщения в messages таблицу                          │
│ • МЕДИА: использует copyMessage API → Storage Channel                       │
│   ✓ Получает file_id из Storage Channel                                     │
│   ✓ Записывает в media_files БД                                             │
│   ✗ НЕ скачивает на диск                                                    │
│                                                                              │
│ ⚡ Параметры:                                                               │
│   - WORKER_CONCURRENCY=3                                                    │
│   - QUEUE_NAME=tasks:osint                                                  │
│   - STORAGE_CHANNEL_ID=-1001234567890  (приватный чат)                     │
│                                                                              │
│ 📁 Хранение медиа (ВАЖНО!):                                                 │
│   Поток: Сообщение с фото → copyMessage → Storage Channel → file_id → БД  │
│   Преимущества:                                                             │
│   ✓ Не используется место на диске                                          │
│   ✓ Файлы остаются на Telegram серверах                                     │
│   ✓ file_id永远有效 (вечный, не истекает)                                   │
│   ✓ Быстрый доступ через Telegram API                                      │
└─────────────────────────────────────────────────────────────────────────────┘

🚀 БЫСТРЫЙ СТАРТ:

1. Создать .env из .env.example
2. Заполнить TELEGRAM_TOKENS (токены ботов)
3. Получить STORAGE_CHANNEL_ID (ID приватного канала)
4. docker-compose up -d
5. Проверить: curl http://localhost:8000/health

# Использование ресурсов

# PostgreSQL: количество сохраненных сообщений
docker-compose exec postgres psql -U nexus_user -d nexus_db \
  -c "SELECT COUNT(*) as message_count FROM messages;"

⚙️ ДОБАВЛЕНИЕ НОВОГО ВОРКЕРА:

1. Создать файл core/workers/my_worker.py
2. Наследовать BaseWorker и реализовать process()
3. Добавить сервис в docker-compose.yml:

    worker-mybot:
      build: .
      command: python -m core.workers.main --worker mybot
      environment:
        - QUEUE_NAME=tasks:mybot

4. docker-compose up -d worker-mybot
